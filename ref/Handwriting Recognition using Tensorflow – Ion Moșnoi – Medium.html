Handwriting Recognition using Tensorflow

Handwriting text is a difficult task because one single form can represent different characters and one character can have many representation. In the example below the ‘v’ from the end of the text could be a ‘r’ or a ‘v’ in different context.

One of mine first project using Tensorflow was a model to recognize handwritten text.

All the code can be found on github.

In order to test the model The IAM dataset was used. One need to register to the site and download a folder with images and with xml files with position of text in the image. The IAM dataset was written by 657 persons, has 1'539 pages, 5'685 sentences, 13'353 lines and 115'320 words.

In order to get the images with text I parsed the xml files and made pairs (image, text). I have used just small letters + blank label. In each image was written one to three words. The size of the image was (36, 96). In order to read the images with their text as input for neural networks I used tfrecords format. I was able to get around 70 files with 200 images each. Ten of the files I used for testing and all the others for training.

Model with slices

First preprocessing step was slant correction. For each image I have rotated the image with a degree, where a is from -45 to 45 with step 3 degree. Then I calculated the following histogram:

m is the column index, h(m) is the number of pixel from the handwriting in the column, delta(m) is the difference between the first and last pixel of handwriting in the current column. H[m] is 1 when there is a whole line in that column. The degree to rotate the image is chosen such that it has the maximum number of column with H[m] = 1 .

The model with slices has as input a tensor with the dimensions [batch-size, number of slices, height of images, width of a window, number of channels].

In order to get as much information from images I used some filtering to get more channels for each image: Sobel, Laplace, Canny, binarization.

The images was spited in slices using a window size of 10 pixels with the step 3 pixels.

Each slice is processed with a cosine transformation in order to contain more information about the center of the image and some information about neighbors.

Now we begin to use Deep Learning, each slice was processed with CNN (Convolutional Neural Network):

For each images, from CNN we get a sequence of features for each time step. So after applying CNN we get a tensor with shape: [batch size, number of slices, features].

Now we can use LSTM that is more advanced type of recurrent neural network to get character prediction for each slice.

In our case the ‘x’ is the features from each slice, the ‘o’ is a vector of probabilities of size length(vocabulary).

After using recurrent neuronal networks we will get a Tensor with shape: [batch size, number of slices, number of characters + blank label]. This has the probability of a character to appear in slice. In order to get probabilities we apply a softmax function.

Because we have for each image a text not a label for each slice we will use continuous temporal classification. It is a way of defining the error of classification that will be derived in order to get the updates for the neural network’s weights.

loss = tf.nn.ctc_loss(targets, logits, seq_len)

where targets is a sparse tensor, logits are the probabilities that a character to be seen at the time t, seq_len is a tensor with shape [batch-size].

I trained the model for 3 hours on Google Clound platform and got an error of 14% on testing set.

I have deployed the model on a Drupal 8 website using Flask.
